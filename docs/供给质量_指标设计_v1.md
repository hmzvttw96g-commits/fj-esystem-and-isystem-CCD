# 功能链②供给质量 指标设计 v1

> 配套 `config/e_caliber_quality.yml` + `scripts/e_supply_quality_pipeline.py`。
> 记录指标设计取舍（标准：所有口径/权重/标准化选择须留痕、可敏感性、不杜撰数据）。

## 1. 这个环节量什么

功能链 E 域 = **供给规模①（数量）+ 供给质量②（水平）**，等权合成 E_index。
②须与①**正交**——不能再量"多少专业/多少招生"（那是①），而要量"供给的**质量层级**"。
代理：**一流本科专业建设点 + 硕博学位授权点**，落在 AI 专业群（C/B/X）内的项目点。

## 2. 为何复合（一流专业 0.6 + 学位点 0.4）

| 成分 | 可得性 | 跨城市变异 | 角色 |
|---|---|---|---|
| 一流本科专业建设点 | 高·公开·广覆盖（福建约670个，计算机类入选最多） | 9 市都有 → 提供横截面变异 | **主成分 0.6** |
| 硕博学位授权点 | 中·稀疏（福州/厦门强、5 市近零） | 仅强市有 → 单用会让弱市恒零 | 增量 0.4 |

单用学位点 → 5 市结构性零 → CCD 零值退化（WP0.5 教训）。单用一流专业 → 丢失"研究型高度"信息。
故复合：一流专业保覆盖、学位点加高度。权重 0.6/0.4 为先验，**pilot 后做敏感性再冻结**。

## 3. 口径分类（两套映射，因学科≠本科专业）

- **一流专业**按**专业名** → C/B/X，复用 `e_caliber_majors.yml`（home_caliber）。严格嵌套 C⊂B⊂X。
- **学位点**按**学科名** → C/B/X，用 `e_caliber_quality.yml.discipline_caliber_map`（学科口径独立维护：
  如"电子信息（专业学位）0854"含计算机技术/软件工程/AI 方向，归 B；学科代码 `code_verified` 冻结前人工核）。
- 某口径 k 的值 = 该口径及更窄口径项目点的**加权累计**（嵌套，与①一致）。

## 4. 权重（先验，待敏感性）

- 复合：first_class 0.6 / degree 0.4。
- 一流专业 tier：**国家级 1.0 / 省级 0.5**。
- 学位点 level：**博士点 1.0 / 硕士点 0.6 / 专业学位点 0.5**。
- 脚本 `wlookup` 容差匹配（"博士"="博士点"、"专业学位"="专业学位点"），人工填表不必拘泥后缀。

## 5. 累计语义（时间维）

一流专业/学位点一经设立**长期有效** → 按 `designation_year` / `active_from_year` 起**累计**到其后各年。
故该环节**近乎时间不变**：贡献横截面水平、几乎不贡献趋势（与 functional_chain.yml temporal 注一致）。
含义：②主要拉开"福厦 vs 其它市"的水平差，趋势动力来自①。

## 6. 标准化选择（关键取舍）

**实现：两成分各自 min-max（按口径分组、跨城市-年）→ 按 0.6/0.4 复合 → 输出 ∈[0,1]。**

- 为何 min-max 而非原始计数直接复合：一流专业点（数十量级）与学位点（个位量级）量纲悬殊，
  直接 0.6×count+0.4×count 会让一流专业淹没学位点，权重失义。各自 min-max 后才可比、权重才有意义。
- 为何**不按人口人均化**：项目点是"能力存量"非"流量人数"，人均化无意义（config 已定 `非 per_capita`）。
- **与 config `normalization: capacity_or_per_institution` 的关系**：v1 采用 min-max 复合（最简、透明、
  解耦两成分量纲）。"按院校数/容量标准化"（quality **密度**）作为**备选**留待敏感性——它度量的是
  "每院校质量密度"而非"质量水平总量"，含义不同，需 pilot 比较后再定。
- ⚠ **循环性警示**：min-max 是面板相对标准化，与下游 CCD 自身的标准化叠加。下游
  `functional_chain_ccd_pipeline` 若对 E 子指标再 min-max，对本值幂等（[0,1]→[0,1]）；
  若改用其它标准化须复核，避免双重压缩。该点 pilot 时验证。

## 7. 数据采集（公开·人工·不杜撰）

一流专业（教育部 2019—2021 三批 + 福建省教育厅公示）、学位点（国务院学位委审核结果 + 各校研究生院）
均为**公开事实数据**，但具体"哪校哪专业哪批次入选/授权"须**逐条照官方名单填**，**严禁凭记忆杜撰**。

流程：
1. `python3 scripts/e_supply_quality_pipeline.py --make-templates`
   → `data/external/e_supply_quality/{first_class_major_points,degree_points}.csv`（预列福建校，gitignore 原料）。
2. 人工照官方名单填：tier=国家级/省级；level=博士点/硕士点/专业学位点；year=批次/设立年。
3. `--first-class <fcm.csv> --degree <deg.csv>` → 落 `data/panel/functional_chain/e_supply_quality_panel.csv`。
4. 跨校区按培养地归市（沿用①的校区规则）；学科代码冻结前人工核 `code_verified→true`。

## 7.5 一流专业数据已采集入库（2026-06-20）

**来源**：用户提供官方复核文件 `福建省AI一流专业官方补齐_学校专业年份_2019_2025_20260620_213419.xlsx`
（每条认定事件带官方 source_url + evidence_text + verification_status=confirmed_official）。
抽 `02_官方认定事件` 67 条（tier：省级43/国家级24；批次：2019×25/2020×17/2021×25）→
`data/external/e_supply_quality/first_class_major_points.csv`（gitignore 原料）。

**升级事件**：12 组同校同专业 省级→国家级（如福建理工计算机 2019省级→2020国家级）。管道
按"该年最新认定 tier 取代、非累加"处理，已加自测断言锁定。

**交叉验证**：管道聚合的加权分（X口径=全部点加权）vs 文件 `04_城市年份汇总` weighted_score，
**54 个城市-年单元 0 不一致** → 升级感知 + 嵌套 + tier 加权逻辑正确。

**口径分歧（1 处，已裁定）**：文件把"数据科学与大数据技术"归 B_basic；项目
`e_caliber_majors.yml` 归 **C_conservative**（AI 核心三专业之一）。本管道用项目口径（home_caliber，
与供给规模一致），故福建技术师范学院/福建江夏学院的数据科学一流点计入 **C**。X 口径总量不受影响
（已验证 0 不一致）。结果：C 口径一流专业极稀疏（仅福州 2 校 2021 起），印证 AI 核心专业获一流
认定尚少——与发展性错配主题一致。

**当前状态**：面板 80/162 单元已填，`data_status=first_class_only(degree_pending)`。复合暂仅含
一流专业成分（0.6 权重）；因下游 CCD 自身 min-max 对此尺度不变，不影响相对结构，学位点到位后补 0.4。
