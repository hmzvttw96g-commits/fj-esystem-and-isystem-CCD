# 数据采集自动化方案 v1.0

> 配套：docs/口径冻结文档_v1.0_草案.md。原则：**参数全读 config、每次运行出 run_id+manifest+采集日志、人工只出现在"复核"环节而非"搬运"环节**。
> 自动化程度标注：🟢 全自动（脚本端到端）｜🟡 半自动（机器干活+人工触发或导出）｜🔴 人工节点（复核/裁定，不可省）

---

## 一、总架构：五条采集管道 + 一套横切质量闸门

```
                    ┌─ E端管道（计划册PDF → E_B/C/X 面板）
                    ├─ I_firm管道（工商数据 → 企业存量面板）
 config/口径配置 ──→ ├─ I_patent管道（专利检索 → 申请量面板）      → 质量闸门 → 候选表
                    ├─ I_job管道（官方岗位+网档回溯，2021—2024）     (Q1—Q5)    → 🔴人工复核
                    └─ F端管道（年鉴+普惠金融指数）                              → 面板入库(run_id)
```

**横切质量闸门（每条管道落面板前必须全过，任一失败阻断下游）：**

| 闸门 | 内容 | 来源教训 |
|---|---|---|
| Q1 完备性 | E端：参照名单（含曾用名）逐校对照，expect_B 学校（不分办学性质，2026-06-12总体规则）零命中自动生成核查单 | WP0.5：厦理工/泉州师院全漏 |
| Q2 跳变检测 | 任何学校/企业/城市级数值 YoY ≥4× 或 ≤0.25× 自动标红挂起 | WP0.5：中医药2020=58.6× |
| Q3 嵌套单调 | 配置层 `caliber_nesting_check.py` + 面板层每单元 X≥B≥C | 结构纪律 |
| Q4 锚点审计 | 标准化 min/max 所在单元须人工签字 | WP0.5：漳州坏锚点 |
| Q5 去重检查 | 同校同专业同批次/同信用代码/同申请号/同岗位公告 重复行检测 | 福州2020=12471 嫌疑 |

闸门以共享模块 `scripts/quality_gates.py`（待建）实现，输出 gate report 进 run_id 目录。

## 二、E 端管道（自动化率最高，材料基本在手）

**现状盘点**：2019（5个批次doc/docx）、2020—2025（PDF）原始文件已在本地（见源表05清单）；缺 2015—2018（决定时间窗起点，见冻结文档待裁定#8）。

| 步骤 | 内容 | 自动化 |
|---|---|---|
| 1. 源文件归档 | 现有文件 sha256 入库登记（仓库不存原文件，存清单+哈希+本地路径）；2015—2018 向省考试院官网/档案回溯，找不到则窗口从2019起 | 🟡 |
| 2. 解析抽取 | pdfplumber（PDF）/ python-docx（doc需先libreoffice转docx）→ 候选行（校名、专业、计划数、批次、页码证据） | 🟢 |
| 3. 归一化 | 校名 join 参照名单（含曾用名表，解决福建工程学院类漏抽）；专业 join 口径配置（代码优先名称兜底）；大类映射 | 🟢 |
| 4. 质量闸门 | Q1/Q2/Q3/Q5 全跑；Q1 零命中校自动生成《真零核查任务单》 | 🟢 |
| 5. 人工复核 | 候选表 reviewer_decision 列（沿用现有 pending→confirmed 流程）；20% 抽样双人编码记 κ；华大/福大晋江校区核验协议 | 🔴 |
| 6. 面板构建 | confirmed 行聚合 → 城市-年份 E_C/E_B/E_X × 公办/含民办 两序列 → run_id 落盘 | 🟢 |

**预计人工量**：每年份约 100—160 候选行（源表已证实量级），6 年 × 复核 ≈ 2—3 人日。

## 三、I_firm 管道

**厦大可用库（2026-06-15 确认）**：CSMAR（国泰安）、CNRDS（正式）、Wind 资讯金融终端（正式）；**无**企查查/天眼查/启信宝等专门工商库。
**⚠ 待登录验证（主路径成立的前提）**：登录 CSMAR 与 CNRDS，确认是否有"工商注册登记/企业注册信息"模块，且字段含 **①成立日期 ②注销/吊销日期 ③行业代码 ④注册地址** 四项——四项齐全则下方主路径成立；缺成立/注销日期则无法做存量重构，转备选路径。

**主路径（🟡，依赖上方验证通过）**：从 CSMAR/CNRDS 工商注册库按"地区=福建9市 × 行业=64/65全部+制造业关键词命中 × 成立日期≤2024"批量导出 CSV。一次性导出，后续全自动：

1. 🟢 去重（统一社会信用代码）→ 剔个体户/分支机构
2. 🟢 存量重构：按成立/注销日期生成 2015—2024 逐年末存量（禁止当前存量倒填）
3. 🟢 关键词分层打标（strong_ai / industrial_digital，读 keywords.yml）→ C/B/X 三套面板
4. 🟢 质量闸门 + 抽样 100 家人工核对经营范围打标准确率 🔴

**备选路径**：CSMAR/CNRDS 无合适工商模块时——国家企业信用信息公示系统（官方，手工分批🟡）或福建省/各市市场监管局公开年报聚合数（精度降为城市级粗口径，标注降级）。

## 四、I_patent 管道

**厦大可用库（2026-06-15 确认）**：incoPat（试用）、智慧芽全球专利检索（CARSI）、壹专利（试用）。**主导出源建议智慧芽（CARSI 相对稳定），incoPat 交叉验证**；试用库宜尽早一次性导全，避免到期补数。
**主路径（🟡）**：智慧芽 / incoPat 检索式批量导出。检索式由脚本从 config 自动生成：

```
(IPC=(G06N OR G06V OR G06F18 OR G06F40 OR G10L15 OR G10L17 OR G06K9...))
AND 申请人地址=(福州 OR 厦门 OR ... 9市) AND 申请日=[20150101,20241231] AND 类型=(发明 OR 实用新型)
```

1. 🟢 检索式生成器（按 C/B/X × 9市 × 年组合，控制单次导出上限）
2. 🟡 平台执行导出（人工触发下载）
3. 🟢 清洗：G06K9 旧码与 G06V/G06F18 去重；第一申请人筛选（高校名单+正则剔除，"有限公司"白名单防误杀）；个人申请人主口径剔除
4. 🟢 city-year × C/B/X 面板 + 质量闸门；🔴 抽样 50 件人工核对申请人类型判定

**备选路径**：国家知识产权局专利检索系统（免费但有验证码，分批手工导出🟡）；Google Patents BigQuery（免费全自动🟢，但申请人地址到地级市映射精度受限，只作交叉验证不作主源）。

## 五、I_job 管道（2021—2024 子样本）

1. 🟢 来源清单驱动爬取：config 维护各市人社局/公共就业网/高校就业网公告列表页 URL 模板，requests+bs4 限速抓取（尊重 robots，间隔≥5s）
2. 🟢 **Wayback Machine 回溯**：对历史已删页面，用 web.archive.org CDX API 自动查询 2021—2024 快照——历史岗位"留存差"问题的主要自动化对策
3. 🟢 关键词分层 + employer_sector 规则引擎预分类（三步判定法第一步自动化）
4. 🔴 三步判定人工复核（v2 协议，双人 20% 抽样）；F_candidate 自动分流落表
5. 🟢 去重 → city-year × C/B/X 面板；商业平台岗位单列 supplement_candidate

## 六、F 端管道（最快，建议先跑——它同时服务 WP0 预检）

1. 🟢 北大数字普惠金融指数：公开 xlsx 下载解析，市级 2011—2023+（免费，不依赖图书馆）
2. 🟢 年鉴变量（存贷款/GDP、金融业工资溢价、就业占比、增加值占比、常住人口、财政科教支出）：**厦大有 EPS数据平台（试用）含《中国城市统计年鉴》库，可按城市×年份直接导出，免抠 PDF**；CNKI 中国年鉴库（CARSI）做备份。取数清单见 `docs/wp0_eps_取数清单_v1.md`
