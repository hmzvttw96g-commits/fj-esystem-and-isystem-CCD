## 一、三口径的总体语义（结构合理性的根基）

C/B/X 在 E、I_firm、I_patent、I_job 四个载体上语义平行，逐层外扩、严格嵌套（C ⊂ B ⊂ X，机器校验）：

| 层 | 语义 | E端（专业） | I_firm（行业×经营范围） | I_patent（IPC） | I_job（关键词） |
|---|---|---|---|---|---|
| **C 保守** | AI 核心 | AI直接命名专业3个 | 64/65行业 ∩ 强AI关键词 | G06N/G06V/G06F18/G06F40/G10L15/17 | 强AI+AI工具岗 |
| **B 基本（主口径）** | AI+数字技术基础 | +计算机大类6专业 | 64/65行业全部 | +G06F/G06Q/G06T/G16Y/H04L | +数字技术岗 |
| **X 扩大** | +智能应用/制造交叉 | +电子信息/自动化/智造11专业 | +制造业数字化交叉企业 | +B25J/G05B/G05D/H04W | +工业数字化岗 |

**三条结构纪律：**
1. **维度正交**：C/B/X 只定义"技术范围"。办学性质（公办/民办）、计划类型（批次）、校区归属、企业存量重构规则、专利申请人规则等是独立维度，各自单独冻结取值，禁止混进 C/B/X——WP0.5 发现的"泉州实为民办口径、厦门实为公办口径"正是维度混杂的事故。
2. **嵌套可机检**：任何口径修订后必须重跑 `caliber_nesting_check.py`；面板层面同时检查计数单调性（每个城市-年份必须 X ≥ B ≥ C，违反即抽取错误）。
3. **统一适用**：三套口径固定适用于所有城市所有年份，禁止城市间动态调权（沿用 I 端升级设计既有纪律）。

## 二、E 端口径（详见 e_caliber_majors.yml）

- **C**（3专业）：人工智能、智能科学与技术、数据科学与大数据技术。*不采用"仅人工智能"方案：该专业 2019 年才首批备案，单专业 C 在早期年份结构性为零，会触发 CCD 零值退化。*
- **B**（9专业+计算机类，主口径）：C + 计算机科学与技术、软件工程、网络工程、信息安全、物联网工程、网络空间安全。与 pilot 既有实践完全一致，保证前后可比。
- **X**（20专业）：B + 电子信息工程、电子科学与技术、通信工程、自动化、机器人工程、智能制造工程、智能感知工程、智能装备与系统、工业智能、区块链工程、大数据管理与应用。
- **大类映射**：计算机类→B；电子信息类→X；自动化类→X；表外大类 manual_review。
- **正交维度取值**：
  - 办学性质：主口径 = 公办本科（含华侨大学等部属）；民办+独立学院单独采集为 E_incl_private 稳健性序列（两序列都建，裁定只影响哪个当主表）。
  - 批次：本科各批次+专项计划计入；高职专科批剔除；征求志愿去重。
  - 校区：按培养地归市；跨校区按最后一年所在地；境外校区剔除。
- **滞后与变换**（框架 v1.1）：主模型用 t-4 滞后；人均化后标准化；留存率调整与永续盘存做稳健性。

## 三、I 端口径

### I_firm（主口径模块1，详见 i_caliber_firm.yml）
C = 64/65 行业 ∩ 强AI关键词；**B = 64/65 行业全部存续企业（主）**；X = B + 制造业∩（强AI∪工业数字化）关键词。
关键规则：存量按成立/注销日期逐年重构（禁止当前存量倒填）；仅法人企业；统一社会信用代码去重；按注册地归市。

### I_patent（主口径模块2，详见 i_caliber_patent.yml）
C = AI 核心 IPC 6 前缀；**B = C + 数字技术 5 前缀（主）**；X = B + 机器人/控制/无线 4 前缀。
关键规则：按申请日计年；发明+实用新型（主）、仅发明（稳健）；第一申请人为企业才计入（高校/科研院所/个人剔除，校企联合看第一申请人）；**G06K9 旧分类号双检去重**（2022 年模式识别分类号迁移，跨年一致性的命门）。

### I_job（稳健性模块，2021—2024 子样本，详见 i_caliber_job.yml）
与既有 I_upgrade_C/B/X 定义对齐：C = 强AI+AI工具岗；B = +数字技术岗；X = +工业数字化岗。
三步边界判定 v2 适用；F 端分流岗位自动落表 F_candidate（论文B佐证变量的采集副产品）。

### I_index 合成（框架 v1.1）
主口径 I_index = 等权(I_firm_B 人均, I_patent_B 人均)；C/X 套同构计算做稳健性。I_job 不进主口径。

## 五、标准化与零值（全口径共用，框架 v1.1）

1. 所有原始量先人均化（除以城市常住人口）再标准化。
2. 标准化用全样本固定基准 min-max；**锚点（min/max 所在单元）必须通过学校/企业级数值审计后人工签字确认**（WP0.5 教训：旧锚点是漳州两个坏值）；秩标准化做稳健性。
3. 任一指数为 0 的城市-年份：D 标记"不可计算"，不参与等级与路径分类。
4. 模块合成权重事前冻结为等权；熵权法仅作披露性稳健性。
