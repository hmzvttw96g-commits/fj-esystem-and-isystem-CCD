# WP0 F变量 · EPS 取数清单 v1

> 目的：登录厦大图书馆 **EPS数据平台**，按本清单导出 9 市（或 41 市扩样）× 2015—2024 的金融变量，
> 回填 `scripts/wp0_fvar_precheck.py` 的输入面板，即可跑"福建9市 or 扩闽浙粤"预检。
> 数据源以《中国城市统计年鉴》为主（EPS 内置库，可按城市×年份直接导出，免抠 PDF）。
> ⚠ 城市统计年鉴口径注意：年鉴 Y 版通常报 Y−1 年数据；导出后按"数据所属年份"对齐，勿按年鉴出版年。

---

## 一、登录与定位

1. 图书馆数据库页 → 搜 `EPS` → 进入 **EPS数据平台**（试用，智能访问/CARSI）。
2. 库选择：优先 **「中国城市统计年鉴数据库」**；部分指标在 **「中国区域经济数据库」**「分省/分地市」库。
3. 通用导出操作：选指标 → 选地区（地级市层级）→ 选年份区间 2015—2024 → 导出 Excel。
4. 全程 🟡 人工导出，🟢 后续清洗/计算交脚本。

## 二、需要导出的原始指标（先导"原料"，比值/溢价交给脚本算）

| # | EPS 原始指标（年鉴表名以平台为准） | 用途 | 单位 |
|---|---|---|---|
| R1 | 年末金融机构人民币各项**存款**余额 | 算 findev_loan_deposit_gdp | 万元/亿元 |
| R2 | 年末金融机构人民币各项**贷款**余额 | 同上 | 万元/亿元 |
| R3 | 地区生产总值 **GDP** | 多个比值的分母 | 万元/亿元 |
| R4 | **金融业**城镇单位从业人员数（分行业就业，"金融业"门类 J） | 算 siphon_fin_emp_share | 人 |
| R5 | 城镇单位从业人员**总数**（全行业合计） | siphon_fin_emp_share 分母 | 人 |
| R6 | **金融业**城镇单位从业人员**平均工资**（分行业平均工资，"金融业"） | 算 siphon_fin_wage_premium | 元 |
| R7 | 城镇单位从业人员**平均工资**（全行业平均） | 工资溢价分母 | 元 |
| R8 | **金融业增加值**（分行业增加值，"金融业"） | 算 fin_value_added_gdp_share | 万元/亿元 |
| R9 | **年末常住人口**（或户籍人口，二选一并全程统一） | 人均化分母（A域共用） | 万人 |
| R10 | 一般公共预算**教育+科技支出** | 控制变量财政科教支出 | 万元 |
| **R11** | **信息传输、软件和信息技术服务业**城镇单位从业人员数（分行业就业，门类 I） | **论文A 产业承接主口径**（顺便导，同表）；人均化后进 I_index | 人 |

> 说明：R1—R10 是 F 端（论文B/WP0）"原料指标"。**R11 是论文A 的产业承接代理**——CnOpenData 全量工商付费不可得后，产业承接改用此年鉴行业规模（全覆盖、免费），见 functional_chain.yml 决策说明。既然在分行业城镇单位就业表里导金融业(R4)，**顺手把信息传输软件业(R11)一起导**，零额外成本。
> R1—R11 是"原料指标"。F 变量比值/溢价由脚本据 R1—R10 计算；产业承接 = R11/常住人口(R9) 标准化。**导原料而非导比值，便于口径复核与留痕**。

## 三、脚本输入面板的 5 个变量怎么算

输入面板列名固定（与 `config/wp0_fvar_precheck.yml` 一致）：

| 面板列名 | 计算式 | 用到的原料 |
|---|---|---|
| `findev_loan_deposit_gdp` | (存款 R1 + 贷款 R2) / GDP R3 | R1,R2,R3 |
| `findev_digital_finance_index` | 北大数字普惠金融指数（市级，**另从北大公开 xlsx 取**，非 EPS） | 北大指数 |
| `siphon_fin_wage_premium` | 金融业平均工资 R6 / 全行业平均工资 R7 | R6,R7 |
| `siphon_fin_emp_share` | 金融业就业 R4 / 城镇单位就业总数 R5 | R4,R5 |
| `fin_value_added_gdp_share` | 金融业增加值 R8 / GDP R3 | R8,R3 |

> `findev_digital_finance_index` 不在 EPS——它来自北京大学数字金融研究中心公开发布的《数字普惠金融指数》（地级市层，2011—2023+），免费 xlsx 下载，按城市×年份并入即可。

## 四、地区范围

- **先导福建 9 市**：福州、厦门、泉州、漳州、莆田、三明、南平、龙岩、宁德。
- 若后续预检判 EXPAND，再补导浙江 11 + 广东 21 市（清单见 `config/wp0_fvar_precheck.yml` 的 `minzheyue_extended`）。EPS 同一界面多选地区即可，扩样几乎零额外成本。

## 五、回填与跑预检

1. 生成空白模板：`python3 scripts/wp0_fvar_precheck.py --make-template`
   → 得到 `city,year,findev_loan_deposit_gdp,findev_digital_finance_index,siphon_fin_wage_premium,siphon_fin_emp_share,fin_value_added_gdp_share` 的空表（福建9×2015—2024）。
2. 把第三节算好的 5 列数值填进去（缺某年某市留空即可，脚本自动跳过）。
3. 跑：`python3 scripts/wp0_fvar_precheck.py --input <填好的panel.csv>`
   → 出 run_id 报告：逐变量组间F/p、ICC、组内CV + EXPAND/FUJIAN_OK 建议。
4. 把报告发我，一起解读、定样本边界（这一步是 WP0 的签字闸门）。

## 六、取数避坑

1. **年份对齐**：城市统计年鉴存在"出版年 = 数据年+1"的错位，按数据所属年份对齐。
2. **口径一致**：金融业就业/工资若分"城镇单位"与"全社会"两口径，全程取同一种（推荐城镇单位，年鉴分行业表的标准口径）。
3. **缺失不填零**：某市某年缺，留空，禁止填 0（WP0.5 零值退化教训）。
4. **常住 vs 户籍人口**：二选一并全程统一；人均化（A域 E/I）须与此一致，建议常住人口。
5. **单位统一**：存贷款/GDP/增加值同量纲（都换算成万元或都亿元）再相除。
